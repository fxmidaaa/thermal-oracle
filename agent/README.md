# ThermalOracle Agent (Windows)

Лёгкий CLI-агент: раз в секунду снимает CPU/GPU температуру и мощность, обороты
вентиляторов и имя активного процесса; складывает в локальный SQLite-спул и раз
в 30 секунд отправляет на бэкенд gzip-батчами. Переживает оффлайн, сон ноутбука
и падения бэкенда — данные не теряются (ретеншн спула: 24 ч).

## Требования

1. **Windows 10/11, Python ≥ 3.11.**
2. **LibreHardwareMonitor** (свежий релиз), запущенный **от администратора** —
   иначе часть сенсоров (EC, SMBus) будет пустой. Сам агент админ-прав не
   требует: Ring-0 читает LHM, агент парсит его JSON.
3. В LHM включить веб-сервер: **Options → Remote Web Server → Run**
   (порт по умолчанию 8085).

## Установка и запуск

```powershell
cd agent
pip install -e .

# 1. Проверить, что сенсоры находятся (dry-run правил выбора):
thermal-agent detect-sensors

# 2. Сопрячь с аккаунтом: pairing-код берётся в дашборде/Swagger
#    (POST /api/v1/devices/pairing-code; код живёт 10 минут, одноразовый)
thermal-agent pair --code XXXX-XXXX --api-url http://127.0.0.1:8000
#    (низкоуровневый обход: thermal-agent register --token to_...)

# 3. Запуск (Ctrl+C — остановка):
thermal-agent run
#    без бэкенда (только локальный сбор в спул):
thermal-agent run --no-ship

# Диагностика: конфиг, доступность LHM, глубина спула:
thermal-agent status
```

Конфиг и спул живут в `~\.thermal_oracle\` (`config.toml`, `spool.db`).
Токен хранится в конфиге открытым текстом (каталог защищён ACL профиля
пользователя); на сервере — только его SHA-256.

## Автозапуск (Планировщик заданий)

```powershell
schtasks /Create /TN "ThermalOracleAgent" /SC ONLOGON /TR "pythonw -m thermal_agent run" /RL LIMITED
```

(LibreHardwareMonitor при этом должен стартовать сам — у него есть собственная
опция автозапуска с правами администратора.)

## Как агент выбирает сенсоры

Дерево LHM различается между вендорами, а числовые id узлов меняются между
перезапусками — поэтому агент не хардкодит имена, а на каждом опросе применяет
правила к свежему дереву: CPU/GPU определяются по иконкам/именам узлов,
`cpu_temp` = максимум по ядрам (исключая «Distance to TjMax»), мощность —
сенсор «Package», `fan_rpm` = максимум по всем вентиляторам. Локализованные
значения («45,5 °C», «3 540 RPM») парсятся корректно. `detect-sensors`
показывает, что именно выбрано.

## Гарантии доставки

- сэмпл сначала пишется на диск (WAL), потом отправляется;
- батч получает `batch_id` один раз и при любых ретраях уходит с тем же id —
  сервер схлопывает повторы (идемпотентность);
- из спула строки удаляются только после `2xx`;
- сеть/5xx → экспоненциальный backoff (1 с → … → 5 мин) с уважением
  `Retry-After`; после оффлайна бэклог дренируется укрупнёнными батчами
  (до 120 сэмплов) без 30-секундных пауз.
