# ADR-0002: Управляемая калибровка — в v2, v1 строго пассивный сбор

Дата: 2026-06-11 · Статус: принято (решение владельца продукта)

## Контекст
Пассивные Rth-точки зависят от рабочей точки (мощность, RPM, тип нагрузки) —
сравнимость между неделями обеспечивается стратификацией и робастной
статистикой, но остаётся шум. Воспроизводимая контролируемая нагрузка даёт
эталонные точки с минимальной дисперсией.

## Решение
В v2: кнопка «Запустить калибровку» в интерфейсе → команда агенту → агент
запускает встроенный CPU-бёрнер с фиксированной мощностью (~30 секунд, не
меньше минимальной длительности стабильного окна) → бэкенд получает эталонную
Rth-точку с пометкой `calibration`. v1 не содержит никакого кода управления
нагрузкой — только пассивный сбор.

## Последствия
- Никаких заделов в коде v1, кроме уже существующих: окна с
  `dominant_process` агента-бёрнера лягут в отдельную страту естественным
  образом; канал «сервер → команда агенту» появится в v2 (потребует
  long-poll/WebSocket в агенте).
- Маркетинговая фича: «нажми кнопку — получи честный замер здоровья охлаждения».
